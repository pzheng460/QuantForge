# 比特币赫斯特-卡尔曼量化交易策略

## 1. 项目概述 (Project Overview)
本项目是一个针对单一资产（比特币 BTC）的量化交易系统，由 Python 实现。
本策略完全摒弃机器学习（Machine Learning）黑盒，转而使用**纯信号处理**和**分形几何约束**。

* **交易者画像：** 个人开发者，Python 技术栈。
* **目标资产：** BTC/USDT (永续合约或现货)。
* **交易频率：** 日内波段 (Intraday Swing)，时间周期 15分钟 (15m)。预计日均交易 1-5 次。
* **核心哲学：** “猎人与狗” —— 使用赫斯特指数判定市场环境（状态），使用卡尔曼滤波追踪真实价值（信号）。

## 2. 数学核心 (Mathematical Core)
本策略依赖两个独立的数学引擎：

### 引擎 A：市场状态识别 (分形几何)
**工具：** 基于重标极差分析 (R/S Analysis) 的赫斯特指数 ($H$)。
**逻辑：**
* **$H < 0.45$：均值回归模式 (弹性)**。价格表现为 Ornstein-Uhlenbeck (OU) 过程。
* **$H > 0.55$：趋势增强模式 (惯性)**。价格表现为分形布朗运动 (fBm)，具有记忆性。
* **$0.45 \le H \le 0.55$：随机漫步 (噪声)**。无序波动，禁止交易。

### 引擎 B：信号提取 (控制理论)
**工具：** 一维卡尔曼滤波器 (1-D Kalman Filter)。
**逻辑：**
* 将观测价格 ($Z_t$) 分离为真实价值 ($X_t$) 和噪声 ($R$)。
* 根据波动率动态调整“卡尔曼增益” ($K$)：波动大时信模型，波动小时信市场。
* 提供“零延迟”的移动均线，优于传统的 SMA/EMA。

## 3. 策略逻辑与规则 (Strategy Logic)

### 全局参数
* **K线周期 (Timeframe)：** 15分钟 (`15m`)
* **Hurst 计算窗口：** 100 周期 (约过去 24-25 小时数据)
* **卡尔曼参数：** $R=0.1$ (测量噪声), $Q=1e-5$ (过程噪声)
* **Z-Score 窗口：** 滚动计算最近 50 个周期的 (Price - KalmanPrice) 差值标准差

### 状态 A：均值回归 ($H < 0.45$)
* **概念：** “橡皮筋原理” —— 交易价格的极端偏离。
* **开多 (Entry Long)：** Price < KalmanPrice - 2.0 * std_dev
* **开空 (Entry Short)：** Price > KalmanPrice + 2.0 * std_dev
* **平仓 (Exit)：** 价格回归穿越 KalmanPrice (Z-Score 回归到 0)。
* **止损：** 固定百分比硬止损 (如 2%) 或 Z-Score > 4.0 (意味着模型失效)。

### 状态 B：趋势跟踪 ($H > 0.55$)
* **概念：** “火车头惯性” —— 顺着偏离方向交易。
* **必要条件：** 卡尔曼均线的斜率 (Slope) 必须与价格方向一致。
* **开多 (Entry Long)：** Price > KalmanPrice 且 Kalman_Slope > 0
* **开空 (Entry Short)：** Price < KalmanPrice 且 Kalman_Slope < 0
* **平仓 (Exit)：** 趋势反转 (斜率变号) 或 $H$ 值跌破 0.5。

## 4. Python 架构指南 (Architecture)

### 依赖库
* `ccxt`: 负责数据获取 (fetch_ohlcv) 和订单执行。
* `numpy`: 负责 R/S 分析和卡尔曼滤波的矩阵运算。
* `pandas`: 负责时间序列管理。
* `asyncio`: 实现非阻塞的数据轮询循环。

### 模块结构
1.  **`MathEngine` 类 (静态计算引擎):**
    * `calculate_hurst(series)`: 输入价格序列，返回浮点数 H。
    * `KalmanFilter` 对象: 维护滤波器的内部状态 (`self.P`, `self.x`)。
2.  **`DataFeed` 类 (数据源):**
    * CCXT 的异步封装。
    * 维护一个长度为 `lookback * 2` 的价格缓存队列 (`deque`)。
3.  **`StrategyCore` 类 (策略核心):**
    * **输入:** 最新价格 (Tick)。
    * **流程:** 更新 Kalman -> 计算 Hurst -> 计算 Z-Score -> 状态切换判断 -> 生成信号。
    * **输出:** 动作指令 (BUY, SELL, HOLD, CLOSE)。
4.  **`RiskManager` 类 (风控):**
    * 每日最大回撤检查 (熔断机制)。
    * 基于凯利公式或固定比例计算每次开仓的大小。

## 5. 开发路线图 (Roadmap)
1.  **回测阶段 (Backtest):** 使用 `vectorbt` 或自定义循环，在 2023-2024 比特币历史数据上验证 $H$ 的有效性和 $Q/R$ 参数的敏感度。
2.  **模拟盘阶段 (Paper Trading):** 在 Testnet 上挂机运行。验证信号延迟、计算稳定性以及 API 连接的健壮性。
3.  **实盘阶段 (Live):** 部署最小仓位进行测试。

## 6. 风控硬约束 (Hard Rules)
* **单日最大亏损：** 总权益的 3% -> 触发熔断 (停止机器人)。
* **单笔最大仓位：** 总权益的 10% (不加高杠杆)。
* **滑点容忍度：** 0.1%。
* **费率考量：** 策略的单笔预期收益必须覆盖 0.1% 的双向手续费成本，否则不予开单。
